✅ Use Case Name

Complete Fraud Risk Assessment (FRA) Sheet

⸻

✅ Description

Allows the Project Manager (PM) to document fraud risk scenarios by completing the FRA sheet using mandatory dropdown fields (Fraud Scheme, Threat Event, Impact Category, Recommended Controls).

The system supports adding multiple rows, provides guidance links, validates inputs in real time, detects duplicate combinations with warnings, and prevents submission until required data is completed.

⸻

✅ Target Output

A fully completed and validated FRA sheet containing one or more fraud risk scenarios, stored as structured records and ready for progression to the next stage of the risk assessment workflow.





✅ Use Case Name

Capture and Manage Risk Treatment Strategy in Fraud Risk Assessment

⸻

✅ Description

This use case allows the Project Manager to document risk treatment strategies within the FRA using a free-text field supported by inline comments and annotations. The system ensures data persistence through auto-save and manual save options, enables export to PDF/Word for sharing, and provides action controls such as submit, save as draft, and clear. Users can also assign action owners, target dates, and statuses to ensure accountability and structured risk mitigation planning.

⸻

✅ Target Output

The risk treatment strategy is successfully recorded with comments, saved for future access, assigned with ownership and timelines, and made available for submission, export, and stakeholder review.






Use Case Name: Verify RFO/Country Coverage Endorsement

Description:
Allows the Project Manager to review the endorsement status provided by RFO/Country Coverage, verify comments, and proceed with offline endorsement if required. The user can upload supporting evidence, select the appropriate endorsement option (e.g., endorse as 1LOD), view current statuses, and navigate between stages before final submission.

Target Output:
RFO/Country Coverage endorsements are verified, required evidence is successfully uploaded, endorsement status is accurately reflected in the system, and the assessment is ready for progression toward final completion.






Use Case Name: Submit Final Endorsed Risk Assessment

Description:
Allows the Project Manager to confirm completion of all required checklist items and formally submit the final endorsed Fraud Risk Assessment. The user verifies prior sections, selects the required confirmation checkbox(es), and clicks on “Complete Final Endorsed Risk Assessment” to finalize the process.

Target Output:
The assessment status is updated to final endorsed/completed state, all checklist confirmations are recorded, and the Fraud Risk Assessment is formally closed for further edits and ready for record or downstream reporting.







*Use Case Name: Automated Email Notification to RFO upon FRA Submission

Description:
When a Project Manager submits the Fraud Risk Assessment for approval, the system automatically triggers an email notification to the assigned Risk Framework Officer (RFO) informing them that an assessment is pending review. A confirmation message is displayed to the Project Manager indicating that the submission was successful and the notification has been sent. The system will also generate reminder and overdue notifications if no action is taken by the RFO within the defined timelines (currently marked as TBD). Notifications reset once the RFO responds (e.g., refer back or endorse).

Target Output:
Email notification successfully delivered to the relevant RFO, confirmation displayed to the Project Manager, and reminder/overdue notifications scheduled with timelines set as TBD until finalized.








*Use Case Name: Automated Email Notification to Country Coverage

Description:
When a Project Manager submits the fraud risk assessment for approval, the system automatically triggers an email notification to the relevant Country Coverage team to inform them that an assessment is pending their review. The system also displays a confirmation message indicating that the assessment was submitted successfully and the notification has been sent. Reminder emails and overdue notifications will follow a defined schedule, with timelines currently marked as TBD, and notifications will reset once a response is received (e.g., Refer back).

Target Output:
Country Coverage receives automated email notifications upon assessment submission, including reminder and overdue alerts based on TBD timelines, ensuring awareness and timely review of pending assessments.







Use Case Name: Review and Action Fraud Risk Assessment (Country Coverage)

Description:
Enables Country Coverage users to view the fraud risk assessment submitted by the Project Manager, review all responses, provide comments, and take action by either endorsing the assessment or referring it back for updates. The system restricts editing, captures feedback in the audit trail, and routes the assessment based on the selected action.

Target Output:
Assessment is either endorsed and forwarded to the Project Manager for final completion or referred back with recorded comments, ensuring traceability and controlled workflow progression.








Use Case Name: Review and Endorse Fraud Risk Assessment by RFO

Description:
This use case enables the Risk Framework Officer (RFO) to review the fraud risk assessment submitted by the Project Manager, evaluate the responses, provide comments, and either endorse the assessment or refer it back for updates. The system captures feedback in the audit trail and routes the assessment based on the RFO’s decision while restricting any edits to the original responses.

Target Output:
The fraud risk assessment is either endorsed and routed to the Project Manager for final completion, or referred back with comments for revision, with all actions logged for audit purposes.







Use Case Name: Initiate Fraud Risk Assessment

Description:
Allows the Project Manager to initiate a Fraud Risk Assessment by defining the scope (group or country), selecting impacted business functions, assigning the relevant RFO, specifying whether the risk applies at the initiative or programme level, and entering required details such as initiative/programme name, IDs, and change description before submitting the request.

Target Output:
A successfully created Fraud Risk Assessment request with all mandatory details captured, routed into the workflow for further assessment and review.

Use Case Name: Access and Manage Fraud Risk Assessment Landing Page
